主管權限檢查

請先在「員工登入」頁面用你的手機號碼登入，再回到這裡按下方按鈕檢查你是否擁有主管權限。

公務機模式

開啟公務機模式後：
員工可以登入自己的帳號，用這台設備來運作排休作業，可允許多人填寫排班資料。
主管可在本設備進行：每月排休鎖定作業，審核員工特休申請作業，設定員工有主管權限。
公務機適合固定放在店裡、大家輪流用的公務手機/iPad。主管作業仍需以主管個人帳號密碼進行登入。

本機待審清單

列出這台裝置上，主管在登入後所收到所有的尚未經主管處理的特休申請。
如為公務機模式的話，可多人輪流登入，用這一台裝置進行排休規劃及特休申請，最後主管可在登入後在這台機器上進行員工填寫的申請單審核工作並將結果送至主機上。

主管權限管理

勾選的員工將可進入主管後台，但是沒有連線至主機的能力，需另外在主管後台自行輸入連線碼。系統管理員帳號永遠擁有主管權限，不受此設定影響。

支援排班登記

當某班別當天人力短缺，可指定另一位當天有上班的員工支援，會顯示在他自己那天的格子上（例如「支A1」），不影響他原本的班別記錄。

點選下方「空白（未排休）」的日期來設定支援日，紅色代表該員工當天已排休/特休，無法設定支援。

目前是檢視模式（唯讀）

這台裝置沒有設定 GitHub Token，可以查看所有資料，但無法核准申請、開通員工或做任何寫入操作。若要正式寫入，請在有設定 Token 的那台裝置（例如公司公務機）上操作。

我的 GitHub Token 設定

此 Token 只會存在你目前使用的這台裝置（瀏覽器 localStorage），不會同步給其他主管，每位主管都要在自己的裝置上各自輸入一次。具備寫入權限的操作（核准申請、開通員工、鎖定月份等）都需要這裡設定好有效的 Token 才能執行。
